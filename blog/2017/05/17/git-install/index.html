Gitインストール

May 17, 2017   #git 

環境とインストールバージョン

  • Windows7 64bit
  • Git for Windows2.13.0

インストール手順

  1. 以下のページのDownloadsから最新版をダウンロードする。
    https://git-scm.com/

  2. ダウンロードしたexeファイルをダブルクリック。

  3. Nextをクリック

  4. インストール先の設定
    デフォルトのままNextをクリック。

  5. インストールするコンポーネントの選択
    すべてデフォルトで問題ありませんが、コンポートネントの詳細を記載します。

    • Additional icons
      デスクトップのショートカットアイコンの設定。使うことはなさそうなのでOFF。

    • Windows Explorer integration
      エクスプローラーのコンテキストメニューの設定。CUIでGitを使う場合はよく使ので必須。ON。

    • Git LFS
      サイズの大きなバイナリファイルを扱うための設定。ON。

    • Associate .git* configration files with the default text editor
      .gitファイルをデフォルトのテキストエディタで起動する設定。ON。

    • Associate .sh files to be run with Bash
      .shファイルをBashで起動する設定。ON。

    • Use a TrueType font in all console Windows
      コマンドプロンプトなどすべてのコンソールのフォントを変更する設定。 ONにすると文字化けすることもあるようなのでOFF。

  6. スタートメニューフォルダの選択
    スタートメニューに追加するフォルダを選択。デフォルトのままNextをクリック。

  7. 環境変数PATHの追加設定
    デフォルトのままNextをクリック。

    • Use Git from Git Bash only
      環境変数PATHには何も追加されず、Git BashでのみGitコマンドを使える。
    • Use Git from the Windows Command Prompt
      環境変数PATHに以下のパスがが追加され、コマンドプロンプトやPowerShellからもGitコマンドを使える。
      • C:\Program Files\Git\cmd
    • Use Git and optional Unix tools from Windows Command Prompt
      環境変数PATHに以下のパスがが追加され、コマンドプロンプトやPowerShellからGitコマンド、Unixコマンドを使える。
      Windowsのfind、sortなどのコマンドが上書きされてしまう。
      • C:\Program Files\Git\cmd
      • C:\Program Files\Git\mingw64\bin
      • C:\Program Files\Git\usr\bin

  8. SSH実行方法の選択
    HTTPSアクセスでOpenSSLかWindowsのSecure Channelのどちらのライブラリを使うかの選択。
    OpenSSLで問題ないのでデフォルトのままNextクリック。

    • Use the OpenSSL library
    • Use the native Windows Secure Channel library

  9. 改行コード変換の設定
    チェックアウト時とコミット時に改行コードをどのように変換するかの設定。

    • Checkout Windows-style, commit Unix-style line endings
      • チェックアウト時->CRLF
      • コミット時->LF
    • Checkout as-is, commit Unix-style line endings
      • チェックアウト時->変換しない
      • コミット時->LF
    • Checkout as-is, commit as-is
      • チェックアウト時->変換しない
      • コミット時->変換しない

    改行コードはエディタで設定するので一番下の変換しないをチェックし、Nextクリック。

  10. Git Bashで使用するコンソールの設定
    Mintty、コマンドプロンプトどちらを使用するかの設定。

    • Use MinTTY
    • Use Windows’ default console Windows
      多くのサイトではMinTTYを選択していますが、私はコマンドプロンプトでも使いやすいので下を選択し、Nextクリック。

  11. その他の設定

    • Enable file system caching
      git status、git logなどのコマンドの応答速度を改善するオプション。
      PCリソースを多少消費する。リソースに余裕はあるのでONのまま。
    • Enable Git Credential Manager
      Git Credential Manager for Windowsをインストールするかのオプション。
      インストールするとHTTPS接続する時のパスワードの入力を省略できる。ONのまま。
    • Enable symbolic links
      シンボリックリンクを使えるようにするかのオプション。特に必要ではないのでOFFのまま。

    Installをクリックする。

  12. 完了
    インストール完了です。
